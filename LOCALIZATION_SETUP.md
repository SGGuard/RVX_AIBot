# 🌐 RVX Bot - Двуязычная локализация (Русский & Украинский)

## ✅ Статус: Полностью настроено и протестировано

Ваше приложение полностью поддерживает локализацию на **русском** 🇷🇺 и **украинском** 🇺🇦 языках с автоматическим переключением при первом использовании.

---

## 📋 Что реализовано

### 1. **Двуязычные переводы**
- ✅ **760 переводов** на русском языке в `locales/ru.json`
- ✅ **760 переводов** на украинском языке в `locales/uk.json`
- ✅ Все критические ключи переведены (меню, кнопки, сообщения об ошибках)

### 2. **Автоматический выбор языка при первом использовании**
**Процесс:**
```
1. Пользователь отправляет /start
2. Бот проверяет, есть ли у пользователя установленный язык
3. Если НЕТ → показывает кнопки выбора языка:
   🇷🇺 Русский  |  🇺🇦 Українська
4. Пользователь нажимает на кнопку → выбранный язык сохраняется в БД
5. Все сообщения бота переключаются на выбранный язык
6. При следующих визитах бот запоминает язык пользователя
```

### 3. **Технические компоненты**

#### `i18n.py` - Модуль интернационализации
```python
# Основные функции:
await get_text(key, user_id=None, language=None, **kwargs)
    # Получает переведённый текст
    
await set_user_language(user_id, language)
    # Сохраняет выбор языка в БД
    
get_user_language(user_id, default=None)
    # Получает язык пользователя
```

#### `locales/` - Директория переводов
```
locales/
├── ru.json    # 760 русских переводов
├── uk.json    # 760 украинских переводов
└── __pycache__/
```

#### `bot.py` - Обработчики команд
```python
# start_command() - Проверяет язык при /start
# handle_language_selection() - Обрабатывает выбор языка
# button_callback() - Обрабатывает все нажатия кнопок
```

#### База данных `rvx_bot.db`
```sql
-- У таблицы users есть колонка:
language TEXT DEFAULT 'ru'  -- Сохраняет предпочитаемый язык
```

## 📝 Структура перевода

Каждый файл перевода имеет формат JSON с иерархической структурой:

```json
{
  "language.select_prompt": "Выберите язык / Select language:",
  "language.russian": "🇷🇺 Русский",
  "language.ukrainian": "🇺🇦 Украинский",
  "activities.lesson1_btn": "💡 Что такое активности?",
  "ask.analyzing": "🤖 Анализирую вопрос...",
  ...
}
```

**Схема именования ключей:**
- `menu.*` - Меню и навигация
- `language.*` - Выбор языка
- `ask.*` - Вопросы к AI
- `activities.*` - Активности и уроки
- `badge.*` - Достижения
- `ai.*` - AI-ответы
- `admin.*` - Администраторские функции
- И другие...

## 🔧 Добавление новых переводов

Если вы добавили новый текст в бота, добавьте ключ в оба файла переводов:

### 1. Добавить в `locales/ru.json`:
```json
{
  "your.new_key": "Ваш русский текст"
}
```

### 2. Добавить в `locales/uk.json`:
```json
{
  "your.new_key": "Ваш украинский текст"
}
```

### 3. Использовать в коде:
```python
text = await get_text("your.new_key", user_id)
await update.message.reply_text(text)
```

## 💡 Советы

1. **Кэширование:** Модуль i18n кэширует переводы и языки пользователей в памяти для быстрого доступа
2. **Fallback:** Если ключ перевода не найден, система вернёт дефолтный текст `[MISSING: key_name]`
3. **Форматирование:** При использовании параметров применяется `.format()`:
   ```python
   await get_text("greeting", name="Иван")  # "greeting": "Привет, {name}!"
   # Результат: "Привет, Иван!"
   ```

## ✨ Что происходит "за кулисами"

```
ПЕРВЫЙ ЗАПУСК:
User → /start → Bot checks DB (no language set)
     → Bot shows language buttons
     → User clicks 🇷🇺 or 🇺🇦
     → handle_language_selection() saves to DB
     → Bot reloads main menu in selected language

ПОВТОРНЫЕ ЗАПУСКИ:
User → /start → Bot checks DB (language found)
     → Bot skips language selection
     → Bot shows main menu in stored language
     → All future messages use stored language
```
